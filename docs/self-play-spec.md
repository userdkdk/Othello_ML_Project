# 셀프 플레이 명세

## 목적
이 문서는 오셀로 학습용 데이터 생성을 위한 셀프 플레이 실행 기준을 정의한다.

## 입력
- 대국 수
- 두 플레이어 정책 또는 에이전트
- 랜덤 시드
- 초기 상태 생성 방식
- 저장 대상 경로 또는 저장 콜백

## 초기 상태 생성 규칙
- 기본값은 `docs/othello-rules.md`의 표준 초기 상태여야 한다.
- 표준 초기 상태 이외의 시작 상태를 허용하려면, 그 상태는 오셀로 규칙을 따라 실제 수순으로 도달 가능한 합법 상태여야 한다.
- 합법 상태의 최소 조건은 다음을 만족해야 한다.
  - 보드 크기와 칸 상태가 규칙 문서와 일치한다.
  - 현재 플레이어가 `BLACK` 또는 `WHITE` 중 하나로 정의된다.
  - 양 플레이어의 돌 개수 차이가 규칙상 도달 가능한 범위를 벗어나지 않는다.
  - 이미 종료된 상태를 초기 상태로 사용하지 않는다. 종료 상태 재생성이 필요하면 별도 데이터셋 생성 작업으로 분리한다.
- 위 조건을 검증할 수 없다면 표준 초기 상태만 허용한다.

## 출력
- 각 대국의 수순 기록
- 상태별 정책 입력 정보
- 최종 승패 또는 보상
- 실행 메타데이터
  - 시드
  - 시작 시각
  - 종료 시각
  - 사용 정책 버전

## 동작 규칙
- 모든 대국은 `docs/othello-rules.md`를 따라야 한다.
- 각 턴에서는 유효 수만 선택할 수 있다.
- 유효 수가 없으면 패스해야 한다.
- 종료 조건과 승패 판정은 엔진 결과를 그대로 사용한다.
- 동일 시드와 동일 정책이면 재현 가능한 실행이 가능해야 한다.

## 데이터 품질 기준
- 상태와 행동은 같은 턴 기준으로 짝지어 저장한다.
- 종료 보상은 대국 종료 후 모든 턴에 일관되게 연결할 수 있어야 한다.
- 보상 기준 관점은 문서나 구현 설정에서 고정되어야 한다. 기본값은 `BLACK` 기준으로 `BLACK 승리 = +1`, `무승부 = 0`, `WHITE 승리 = -1`이다.
- 불완전한 대국 기록은 학습 데이터로 저장하지 않는다.
- 저장 포맷은 후처리 없이 학습 파이프라인이 읽을 수 있을 정도로 명확해야 한다.

## 실패 처리
- 단일 대국 실패와 배치 전체 실패를 구분한다.
- 실패 원인과 시드를 기록한다.
- 부분 저장을 허용할 경우, 완료된 대국과 실패한 대국을 구분 가능해야 한다.
