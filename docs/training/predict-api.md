# 예측 API 계약

## 목적
이 문서는 오셀로 모델 또는 정책 모듈이 엔진과 상호작용할 때 따를 입력/출력 계약을 정의한다.

## 입력
- 현재 보드 상태
- 현재 플레이어
- 유효 수 목록
- 선택적으로 추가 컨텍스트
  - 탐색 온도
  - 탐색 깊이
  - 배치 차원 입력 텐서

## 출력
- 수별 점수 또는 확률 분포
- 선택된 수 또는 패스 결정
- 선택적으로 상태 가치 추정치

## 행동 표현 규약
- 좌표 기반 행동은 `(row, col)` 형식을 사용한다.
- `row`, `col`은 `0..7` 범위의 정수다.
- 패스는 좌표와 구분되는 별도 리터럴 값 `PASS`로 표현한다.
- 엔진과 모델은 동일한 행동 표현을 사용해야 하며, 중간 계층에서 암묵적 변환을 가정하지 않는다.

## 분포 표현 규약
- 기본 분포 정의역은 전체 액션 공간 `65`개다.
  - `(0,0)`부터 `(7,7)`까지 64개 좌표
  - `PASS` 1개
- 유효 수가 없는 경우에만 `PASS`가 선택 가능하다.
- 전체 액션 공간 분포를 반환하면 유효하지 않은 행동의 확률은 `0`이거나 후처리에서 `0`으로 마스킹 가능해야 한다.
- 구현이 유효 수 집합에 대한 압축 분포를 반환한다면, 반환 순서가 입력 `유효 수 목록` 순서와 정확히 일치한다고 별도 명시해야 한다.

## 계약 조건
- 예측 결과는 유효 수 집합과 정합해야 한다.
- 유효하지 않은 수에 점수를 내더라도 최종 선택 결과는 유효 수만 반환해야 한다.
- 패스는 유효 수가 없을 때만 선택 가능해야 한다.
- 입력 보드 상태를 변경하면 안 된다.

## 검증 포인트
- 빈 유효 수 목록에서 패스를 반환하는지
- 단일 유효 수만 있을 때 그 수를 반환하는지
- 확률 분포를 반환하면 합이 1인지 또는 정규화 가능 상태인지
- 배치 예측과 단건 예측의 의미가 일관적인지
