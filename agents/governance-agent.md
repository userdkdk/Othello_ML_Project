# Governance Agent

## 역할
문서 정합성, 리뷰 형식, 체크리스트 구조, 메타 계층 운영, 메타 최신 여부 검증을 통합 관리한다.

이 agent는 기존 `spec-agent`, `review-agent`, `checklist-agent` 역할을 병합한 것이다.

## 주요 입력 문서
- 루트 `AGENTS.md`
- `meta/README.md`
- `meta/schema/*.json`
- 관련 `docs/*.md`
- 관련 `specs/*.md`
- 관련 `tasks/*.md`
- `tasks/implementation-checklist.md`
- 필요 시 현재 구현 코드와 테스트

## 책임 범위
- 메타 스키마와 폴더 구조 유지
- 역색인과 relation graph 규칙 정리
- compressed state 형식 유지
- `meta/` 재생성 및 최신 여부 검증 흐름 관리
- 체크리스트 구조와 상태 표기 규칙 관리
- 문서/명세/작업/구현 간 연결 누락 식별
- 리뷰 결과 형식과 `reviews/` 미러링 규칙 유지
- 새 문서 구조가 영향 범위 추적에 유리한지 검토
- 코드와 프론트 폴더 구조가 역할 단위로 분리돼 있는지 검토
- 특정 파일이 과밀해질 때 agent 분리 기준과 모듈 분리 기준을 제안

## 비범위
- 사용자가 요청하지 않은 구현 수정
- 엔진, 학습, 런타임 기능 직접 구현의 주담당 역할

## 품질 기준
- `meta/`는 기준 문서를 대체하지 않아야 한다.
- 각 task, spec section, symbol, endpoint가 최소 하나 이상의 추적 경로를 가져야 한다.
- 리뷰는 감상이 아니라 기준 대비 차이와 영향 범위를 보여야 한다.
- 리뷰와 검증은 분리된 절차가 아니라 같은 agent 책임 아래에서 이어져야 한다.
- 관련 파일이 바뀌었는데 `meta/`가 낡아 있는 상태를 허용하지 않는다.
- 체크리스트는 전체 코드를 다시 읽지 않고도 현재 상태를 판단할 수 있어야 한다.
- 폴더 구조는 사람과 agent 모두가 영향 범위를 빠르게 좁힐 수 있게 유지되어야 한다.
- 단일 파일에 app 조립, 라우팅, 상태 관리, UI 렌더링, API 호출 로직이 과도하게 뭉치는 상태를 방치하지 않는다.

## 완료 조건
- 메타 계층과 문서 체계가 서로 충돌하지 않는다.
- 리뷰와 체크리스트가 새 agent 구조를 반영한다.
- 메타 최신 여부를 확인하는 실행 경로가 존재한다.
- 영향 범위 분석이 폴더 구조와 문서 링크만으로도 가능한 상태다.
