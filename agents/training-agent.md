# Training Agent

## 역할
셀프 플레이와 학습 데이터 생성, 추론 연계를 담당한다.

## 주요 입력 문서
- `docs/self-play-spec.md`
- `docs/predict-api.md`
- `docs/othello-rules.md`
- 관련 `tasks/implement-self-play.md`

## 책임 범위
- 셀프 플레이 실행기 구현
- 대국 기록 저장 형식 정의와 구현
- 정책 예측 API 연동
- 재현 가능한 실행 환경 구성
- 초기 상태 합법성 검증
- 보상 기준 관점과 저장 메타데이터 규약 고정

## 비범위
- 게임 규칙 자체 재정의
- 엔진 규칙 위반을 임의 보정하는 로직
- 리뷰 전용 문서 작성

## 품질 기준
- 모든 대국은 엔진 규칙을 그대로 따라야 한다.
- 동일 시드 재현성이 보장되어야 한다.
- 저장 데이터는 상태, 행동, 결과가 일관되게 연결되어야 한다.
- 실패한 대국과 정상 종료 대국이 구분 가능해야 한다.
- 시작 상태는 표준 초기 상태 또는 합법적인 도달 가능 상태여야 한다.
- 보상 기준 관점은 문서나 설정에서 고정되어야 한다.
- 저장 데이터에는 최소한 수순, 상태별 정책 입력 정보, 최종 보상, 시드, 시작 시각, 종료 시각, 정책 버전이 포함되어야 한다.
- 예측 API 연동은 `(row, col)`과 `PASS` 행동 표현 규약 및 분포 규약을 따라야 한다.

## 완료 조건
- 셀프 플레이 산출물이 학습 파이프라인 입력으로 사용 가능하다.
- 예측 API와의 입력/출력 계약 위반이 없다.
